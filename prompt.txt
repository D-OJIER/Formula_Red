FORMULA RED A Professional Daily F1-Style Racing League for Reddit
(Devvit)

Inspired by Formula One and built for Reddit.

  --------------------------------------------------
  1. OVERVIEW Formula Red is an asynchronous
  professional racing league where a new track is
  generated every day, players submit one official
  run, podium winners are crowned daily, and a
  season championship runs continuously.
  --------------------------------------------------
  2. DESIGN GOALS - Professional motorsport feel -
  Daily ritual gameplay - Fair and deterministic
  simulation - Scalable to large communities -
  Long-term competitive identity

  --------------------------------------------------

3.  CORE PILLARS

-   One global race per day
-   Same track and rules for all players
-   One official attempt per race
-   Deterministic backend simulation

  --------------------------------------------------
  4. DAILY FLOW - Track generated at UTC day start -
  Practice sessions open - Main race opens after
  practices - Race closes at 23:59 UTC - Podium and
  points awarded
  --------------------------------------------------
  5. DRIVER CONFIGURATION Car Setup: - Downforce:
  Low, Balanced, High - Gear Bias: Acceleration,
  Neutral, Top Speed - Tyres: Soft, Medium, Hard

  Driving Style: - Aggressive - Balanced -
  Conservative

  One Tactical Action: - ERS boost - Defensive
  positioning - Risky overtake - Tyre preservation
  --------------------------------------------------

6.  TRACK SYSTEM Track attributes:

-   length
-   corner density
-   straight length
-   overtaking difficulty
-   tyre degradation
-   weather probability

Track types: - Street - High-speed - Technical - Endurance - Mixed

Track seed: YYYYMMDD

  --------------------------------------------------
  7. DAILY MODIFIER One per day: - Rain - Dirty
  air - High tyre wear - Safety car risk - Low grip
  --------------------------------------------------
  8. SIMULATION MODEL

  lapTime = baseTrackTime + setupImpact +
  drivingStyleRisk + tyreDegradation +
  tacticalActionEffect + trackModifierEffect +
  rng(trackSeed + userId)
  --------------------------------------------------

9.  LEADERBOARDS

Daily leaderboard: - Reset every day - Frozen after race close

Season leaderboard: - Cumulative points - Podiums and participation
tracked

  --------------------------------------------------
  10. POINT SYSTEM
  --------------------------------------------------
  11. SEASONS - Fixed-length seasons (example: 30
  days) - Champion declared at season end - New
  season starts automatically

  --------------------------------------------------

12. DATA MODEL

Daily race store: race:YYYYMMDD - trackSeed - trackConfig - modifier -
results - frozen

Result entry: - userId - lapTime - setup - style - action - timestamp

Season store: season:current - standings - podiums - racesPlayed

  --------------------------------------------------
  13. DEVVIT ARCHITECTURE - Track briefing panel -
  Setup and strategy form - Session selector -
  Leaderboards - Season standings panel
  --------------------------------------------------
  14. FAIRNESS & SECURITY - One official run per
  user - Deterministic RNG - No reruns - Race freeze
  enforced

  --------------------------------------------------

15. SOCIAL INTEGRATION

-   Comment discussion per race
-   Automatic podium announcement
-   Season summary posts

  --------------------------------------------------
  16. SCALABILITY - Constant time simulation -
  Incremental leaderboard updates - No real-time
  dependency
  --------------------------------------------------
  17. OPTIONAL COSMETICS - Liveries - Helmets -
  Badges - Podium emotes

  --------------------------------------------------

18. MVP SCOPE

-   Daily race creation
-   Track generator
-   Practice sessions
-   Main race
-   Leaderboards
-   Podiums
-   Season standings

  --------------------------------------------------
  19. PRACTICE SESSIONS (P1 – P4)

  Each race day includes four official practice
  sessions.

  Sessions: P1 – Track discovery P2 – Setup
  refinement P3 – Strategy testing P4 – Final
  rehearsal

  Timeline example: P1 00:00 – 04:00 P2 04:00 –
  08:00 P3 08:00 – 12:00 P4 12:00 – 16:00 Race 16:00
  – 23:59

  Rules: - One run per user per practice - Practice
  results do not affect podiums or points

  Practice simulation: practiceLapTime =
  baseTrackTime + setupImpact + drivingStyleRisk +
  tyreDegradation + tacticalActionEffect +
  trackModifierEffect + rng(trackSeed + userId +
  sessionId)

  Practice includes small telemetry noise. Main race
  uses clean simulation.

  Practice data keys: practice:YYYYMMDD:P1
  practice:YYYYMMDD:P2 practice:YYYYMMDD:P3
  practice:YYYYMMDD:P4
  --------------------------------------------------

20. IDENTITY

Formula Red is designed as a professional daily racing league inside
Reddit. One track. One race. One official attempt. One podium.
